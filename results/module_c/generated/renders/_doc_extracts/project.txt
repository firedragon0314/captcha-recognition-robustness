2. Brief Introduction of This Project
Text-based CAPTCHA is a common human verification method used to distinguish human users from automated programs. CAPTCHA images usually contain visual distortions such as noise, blur, interference lines, rotation, overlapping characters, and complex backgrounds. These distortions make CAPTCHA recognition a challenging computer vision and sequence-learning problem.
In this project, we study text-based CAPTCHA recognition in a controlled experimental environment. Our goal is not to attack or bypass real-world systems, but to compare different deep learning recognition strategies under corrupted CAPTCHA conditions.
Instead of building only one recognition model, this project compares two fundamentally different CAPTCHA recognition approaches.
1. Sequence-based recognition
2. Position-wise character classification
The project also studies whether image restoration preprocessing can improve recognition robustness under noisy and distorted conditions.
Research Question: This project investigates whether image restoration preprocessing improves CAPTCHA recognition robustness under corrupted conditions, and compares two recognition strategies: CRNN-based sequence recognition and CNN-based position-wise character classification.
The project is organized around three sub-questions:
- RQ1: Does image restoration improve recognition accuracy under corrupted CAPTCHA conditions<->
- RQ2: Which recognition strategy is more robust: CRNN sequence recognition or CNN multi-head character classification<->
- RQ3: Does contrastive CNN feature learning improve robustness under corrupted CAPTCHA conditions<->
The project is divided into three main modules:
- CAPTCHA corruption generation and image restoration
- Sequence-based CAPTCHA recognition
- Robust character-level CAPTCHA classification
吳睿嬛 focuses on CAPTCHA corruption generation and image restoration using Denoising Autoencoder and DnCNN. 葉俊廷 focuses on sequence-based CAPTCHA recognition using CRNN with CTC Loss together with contrastive CNN feature learning. 盛正璿 focuses on robust character classification using a CNN multi-head classifier with contrastive CNN feature learning.
The purpose of this project is to compare how different recognition strategies behave under clean, corrupted, and restored CAPTCHA conditions.
The main contribution of this project is a controlled robustness comparison between sequence-based CAPTCHA recognition and position-wise character classification, with systematic evaluation of whether image restoration improves recognition under corrupted CAPTCHA conditions.
3. Related Work
3.1 Denoising Autoencoder
Pascal Vincent, Hugo Larochelle, Yoshua Bengio, and Pierre-Antoine Manzagol, "Extracting and Composing Robust Features with Denoising Autoencoders," ICML 2008, is the classical reference for Denoising Autoencoder methods. This paper is relevant because our restoration module also learns to reconstruct clean CAPTCHA images from corrupted inputs.
3.2 DnCNN
Kai Zhang, Wangmeng Zuo, Yunjin Chen, Deyu Meng, and Lei Zhang, "Beyond a Gaussian Denoiser: Residual Learning of Deep CNN for Image Denoising," IEEE Transactions on Image Processing, 2017, is a standard residual denoising baseline. We cite it because our project compares autoencoder-based restoration with residual CNN-based denoising before CAPTCHA recognition.
3.3 CRNN + CTC
Baoguang Shi, Xiang Bai, and Cong Yao, "An End-to-End Trainable Neural Network for Image-Based Sequence Recognition and Its Application to Scene Text Recognition," IEEE TPAMI 2016, is the main CRNN reference used in this proposal. It is relevant because our sequence-recognition branch also follows the CNN-RNN-CTC idea for unsegmented text prediction.
3.4 SimCLR
Ting Chen, Simon Kornblith, Mohammad Norouzi, and Geoffrey Hinton, "A Simple Framework for Contrastive Learning of Visual Representations," ICML 2020, provides the SimCLR framework. We use this reference because contrastive learning is part of the CNN feature-learning design in our recognition models.
3.5 MoCo
Kaiming He, Haoqi Fan, Yuxin Wu, Saining Xie, and Ross Girshick, "Momentum Contrast for Unsupervised Visual Representation Learning," CVPR 2020, provides the MoCo framework. It is cited for the same reason as SimCLR: to support robust contrastive CNN feature learning under multiple corrupted CAPTCHA conditions.
4. Dataset Specification
The CAPTCHA dataset used in this project will be generated using a custom Python CAPTCHA generation script based on the `captcha` Python package (`captcha.image.ImageCaptcha`). Each image contains a fixed-length sequence of 5 characters composed of uppercase English letters (A-Z) and digits (0-9).
The CAPTCHA images are generated using the following settings:
- Image size: 160 x 60
- CAPTCHA length: 5 characters
- Character set: uppercase letters and digits
- Random seed: 3027
The dataset generation pipeline is implemented in Python.
A total of 100,000 clean CAPTCHA images will be generated.
Dataset split: 70% training set, 15% validation set, and 15% testing set.
The data preparation process is divided into three stages:
- Stage 1: Generate clean CAPTCHA images using captcha.image.ImageCaptcha.
- Stage 2: Apply additional corruption functions to create corrupted validation and test conditions such as Gaussian noise, blur, rotation, interference lines, background clutter, and partial occlusion.
- Stage 3: Use the restoration models to generate restored CAPTCHA images from the corrupted images.
Therefore, the original Python generation script is responsible for creating the clean CAPTCHA dataset only. Corrupted sets and restored sets are created in later processing stages rather than directly produced by the base generation script.

The synthetic CAPTCHA generation approach provides several advantages for this project.
- Large-scale data generation
- Automatically generated labels
- Controlled corruption settings
- Flexible robustness evaluation
Both 葉俊廷 and 盛正璿 will use the same clean set, corrupted conditions, and restored outputs to ensure fair comparison between sequence-based recognition and position-wise character classification.

5.3 Module 3 - Position-wise Robust Character Classification
This module will be mainly handled by 盛正璿.
The purpose of this module is:
1. Predict each character position with a CNN-based multi-head classifier
Unlike 葉俊廷, which focuses on sequence-based recognition, 盛正璿 focuses on position-wise character classification.
This design is suitable for fixed-length CAPTCHA recognition and aims to improve robustness under noisy and distorted conditions.
The classification pipeline is:
Contrastive CNN Feature Learning
Contrastive CNN feature learning will be used to learn stable CAPTCHA feature representations.
Different augmented versions of the same CAPTCHA image should have similar feature representations, while different CAPTCHA images should remain distant in feature space.
盛正璿 first trains the CNN encoder using contrastive CNN feature learning, where different corrupted versions of the same CAPTCHA image are treated as positive pairs. After pretraining, the encoder is connected to five classification heads, and the full model is fine-tuned for position-wise character prediction.
Different augmentations include:
- Noise
- Blur
- Rotation
- Brightness changes
- Interference lines
Robustness Evaluation
盛正璿 mainly studies:
- Character-level robustness
- Feature stability under corruption
- Generalization ability under distortion
The model will be evaluated under clean, noisy, blurred, distorted, and restored CAPTCHA conditions.
The CNN classifier developed by 盛正璿 is independent from the CRNN sequence model developed by 葉俊廷, and both models will later be compared.
